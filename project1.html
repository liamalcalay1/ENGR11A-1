Project 1

Nametag and Failed Print

Project Description:

The goal of this project was to introduce us to 3D printing and the limits of the printer.

First Print
Nametag

I first downloaded the file for the name tag, which I was then able to edit in PrusaSlicer. I first attempted to modify the name tag by changing the angle of my first and last name to create a v shape, however, I accidentally raised one of the names above the base of the name tag, which didnâ€™t allow the print to be downloaded. I then decided to use the default name tag, which was able to be sliced and then exported. I inserted the USB with the downloaded file into the PrusaSlicer and began printing. Eventually, the printer stopped printing even though the print was not completed, which required me to change the filament color. After changing the filament, the print was able to complete successfully. Overall the print was pretty simple and did not require much time or trouble shooting.

Second Print
Failed Anubis

I decided to print Anubis. The initial file I originally downloaded included supports, so I found a new file without the supports. I found that PrusaSlicer would build in supports, which I had to manually remove. The issue that I believed would arise was that the snout of Anubis was a large overhang that would likely collapse, cause a blob, or create a stringy mess. The base of the head printed perfectly, but issues began once the snout was being printed. The snout ended up being a bit of a stringy mess, which was expected, and the ears also had many small PLA strands between them. Overall the print was a successful failure as the overhang of the snout led to a stringy mess and partial collapse, but the print was still able to reach completion without any damage to the printer. Additionally, I learned that 3MF files store more information about the print including color and texture compared to STL files, which only stores the surface geometry of the 3D model.
